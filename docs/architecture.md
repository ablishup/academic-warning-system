# 学业预警系统 - 系统架构总览

## 一、项目概述

学业预警系统是一个面向高校的学生学业风险管理平台，通过收集多维度学习数据（考勤、视频观看、作业成绩、考试成绩等），运用机器学习算法（随机森林模型）预测学生期末挂科风险，为教师、辅导员提供早期干预依据。

### 1.1 核心功能

| 功能模块 | 说明 | 目标用户 |
|---------|------|---------|
| 数据采集 | 学习活动、作业、考试成绩导入 | 教师 |
| 风险预测 | 基于多维度数据的挂科风险预测 | 系统自动化 |
| 预警管理 | 查看预警学生列表、风险等级分布 | 辅导员 |
| 干预记录 | 记录对学生的帮扶措施及效果评估 | 辅导员 |
| 教学资源 | 上传/下载课程相关资料 | 教师/学生 |
| 学情分析 | 查看个人学习数据可视化 | 学生 |

### 1.2 业务数据流

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 教师上传  │───►│ 数据整合  │───►│ 模型预测  │───►│ 生成预警  │───►│ 辅导员干预│
│ 学习数据  │    │ 多维度分析│    │ 风险等级  │    │ 学生列表  │    │ 记录跟踪  │

                                              │
                                              ▼
                                        ┌──────────┐
                                        │ 学生查看  │
                                        │ 个人学情  │
                                        └──────────┘
```

## 七、关键设计决策

### 7.1 为什么使用 IntegerField 替代 ForeignKey

由于 `Class` 和 `Student` 表设置为 `managed = False`（遗留/外部表，由其他系统维护），Django 的 `ForeignKey` 会尝试创建数据库级外键约束，可能导致迁移问题。因此系统使用 `IntegerField` 存储关联 ID，在应用层维护关系。

| 表 | 字段 | 关联目标 | 实现方式 |
|---|------|---------|---------|
| classes | counselor_id | auth_user.id | IntegerField |
| students | class_id | classes.id | IntegerField |
| courses | teacher_id | teachers.user_id | IntegerField |
| course_enrollments | student_id | students.id | IntegerField |

### 7.2 双认证机制的设计

系统同时支持 JWT 和 Session 两种认证：

- **JWT（主要）**：用于常规 API 调用，无状态、适合前后端分离
- **Session + CSRF（备用）**：用于文件上传等需要 Django 表单验证的场景

这种设计兼顾了现代前后端分离架构和传统 Django 安全机制。

### 7.3 响应格式统一

后端返回两种格式，前端通过 Axios 拦截器统一处理：

| 来源 | 原始格式 | 处理后格式 |
|------|---------|-----------|
| 自定义 API | `{code, data, message}` | 直接返回 |
| DRF 分页 | `{count, results, next, previous}` | 包装为 `{code: 200, data: {...}}` |

这使得前端组件可以用统一的 `if (res.code === 200)` 判断请求是否成功。

### 7.4 风险等级权重设计

综合得分计算公式：

```
综合得分 = attendance_rate * 0.30
        + video_progress * 0.20
        + homework_avg * 0.30
        + exam_avg * 0.20
```

权重依据：
- 出勤率 (30%)：反映学习态度和课堂参与度
- 作业成绩 (30%)：反映日常知识掌握程度
- 考试平均分 (20%)：反映阶段性学习效果
- 视频进度 (20%)：反映自主学习能力

### 8.1 模型算法可替换

`WarningPredictor` 采用接口化设计，核心方法为：

```python
class WarningPredictor:
    def train(self, training_data) -> dict      # 训练模型
    def predict(self, features_dict) -> dict     # 预测风险等级
```

如需替换为其他算法（如 XGBoost、神经网络），只需实现相同接口即可。

### 8.2 数据源可扩展

数据导入模块支持多种格式：
- 当前：Excel (.xlsx) 文件导入
- 可扩展：CSV 导入、API 对接（教务系统）、数据库直连

### 8.3 前端路由可扩展

路由配置采用角色分组，新增角色只需：
1. 在 `router/index.js` 中添加路由组
2. 创建对应的 Layout 组件
3. 在 `views/` 下创建页面目录
