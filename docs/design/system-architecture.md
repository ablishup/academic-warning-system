## 1. 架构设计选择

### 1.1 选定方案：分层数据融合架构

**决策时间**：2026-04-05  
**决策理由**：
- 中期答辩时间紧张，此方案实现更快
- 知识点关联等功能在统一数据层更容易实现
- 教师端只需"导入"功能，无需对接真实API
- 架构清晰，答辩时易于讲解

**架构图**：
```
┌─────────────────────────────────────────────────────────────┐
│                        展示层（四端）                         │
│  辅导员端(核心)    教师端(上传)    学生端(查看)    管理员端     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      业务逻辑层                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │  预警引擎    │  │  数据融合    │  │  知识点关联分析      │ │
│  │  随机森林    │  │  多维度加权  │  │  薄弱点追溯          │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      数据整合层                              │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐            │
│  │ 学习通导入 │  │ 智慧树导入 │  │ 系统自采集 │            │
│  │  (Excel)   │  │  (Excel)   │  │ (视频/作业)│            │
│  └────────────┘  └────────────┘  └────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

**备选方案**：插件式扩展架构（因实现复杂度高而放弃）

## 4. 关键约束

1. **时间约束**：中期答辩在即，优先保证核心功能可用
