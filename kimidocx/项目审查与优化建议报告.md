## 二、🔴 严重问题（必须优先修复）

### 1. CORS 仍允许所有来源

**`settings.py` 第 194 行**：
```python
CORS_ALLOW_ALL_ORIGINS = True
```

虽然这方便了本地开发，但生产环境必须限制为前端域名。

**建议**：
```python
CORS_ALLOW_ALL_ORIGINS = False
CORS_ALLOWED_ORIGINS = os.environ.get('CORS_ALLOWED_ORIGINS', 'http://localhost:8080').split(',')
```

---

### 2. `ALLOWED_HOSTS` 为空

**`settings.py` 第 33 行**：
```python
ALLOWED_HOSTS = []
```

Django 生产环境要求明确配置允许的主机名，空列表会导致非 `localhost` 请求被 400 拒绝。

**建议**：
```python
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', 'localhost,127.0.0.1').split(',')
```

---

### 3. 数据脚本仍硬编码数据库密码

**`data/scripts/import_to_mysql.py` 第 30 行**：
```python
'password': 'YYXyyx204821',
```

虽然主后端已改用 `.env`，但数据导入脚本仍独立硬编码密码，形成安全盲区。

**建议**：数据脚本同样读取 `.env` 或接受命令行参数传入密码。

---

### 4. 数据脚本仍有裸 `except`

**`data/scripts/import_to_mysql.py`** 中存在 5 处裸 `except:`（第 232、414、433、514、532 行）：

```python
try:
    cursor.execute(...)
except:
    pass
```

任何 SQL 错误都会被静默吞食，无法获知导入失败了多少条数据。

**建议**：
```python
except Exception as e:
    print(f"导入失败: {e}, 数据: {row}")
```

---

### 5. `.env` 中第三方 API 密钥暴露

`.env` 文件中包含：
```
DEEPSEEK_API_KEY=sk-dff0b4d93133456ab56c05943b817316
```

虽然 `.env` 已被 `.gitignore` 保护不会进入版本控制，但该密钥属于生产凭证，需提醒用户**不要在任何公开场合分享此文件**。

## 三、🟡 架构与工程化问题（中期优化）

### 6. 后端视图风格仍混乱

各应用视图编写方式统计：

| 应用 | 使用风格 |
|---|---|
| `classes` | `@api_view` |
| `users` | `@api_view` |
| `algorithm` | `APIView` |
| `teacher_dashboard` | `APIView` |
| `courses` | `@api_view` + `generics` |
| `import_app` | `APIView` + `generics` |
| `interventions` | `APIView` + `generics` |
| `learning` | `APIView` + `generics` |
| `warning_system` | `APIView` + `generics` |

同一项目内混用函数视图、类视图、通用类视图，导致接口风格不一致，增加维护成本。

**建议**：新接口统一使用 `ModelViewSet` + `Router`，旧接口逐步迁移。

---

### 7. `get_student_from_user()` 仍重复

该辅助函数仍在以下 3 个文件中重复定义：
- `courses/views.py`（第 230 行）
- `learning/views.py`（第 27 行）
- `warning_system/views.py`（第 26 行）

**建议**：提取到 `users/utils.py` 或新建 `common/utils.py`。

---

### 8. 空壳应用与空壳测试

| 项目 | 状态 |
|---|---|
| `teacher_dashboard/models.py` | 空壳（仅默认注释，60 字节） |
| `back/backend/*/tests.py`（9 个） | 空壳（仅默认注释） |
| `test_prediction.py` | 存在，但需确认是否有实际内容 |

**建议**：删除 `teacher_dashboard/models.py` 或补充模型；为核心业务编写至少 1-2 个真实接口测试。

---

### 9. 前端 `request.js` baseURL 硬编码

```javascript
baseURL: 'http://localhost:8000/api',
```

这导致部署到服务器后所有 API 请求仍指向本地。

**建议**：改为从环境变量读取：`import.meta.env.VITE_API_BASE_URL`，开发环境配置 `.env.development`。

## 四、🟢 优化建议（提升体验）

### 10. 前端调试代码与 mock 数据

- **`console.log` / `console.error`**：**57 处**
- **mock / 模拟数据**：**11 处**
- **`localhost:8000` 硬编码**：`Resources.vue`、`UploadData.vue` 中直接写死上传地址

**建议**：全局搜索 `console.log` 并删除；mock 数据仅在 `NODE_ENV === 'development'` 时启用；上传地址使用相对路径或环境变量。

---

### 11. `/admin/system` 有菜单无页面

`AdminLayout.vue` 中存在"系统设置"菜单项指向 `/admin/system`，但：
- 路由表无此路由
- `views/admin/` 目录下无 `System.vue`

**建议**：要么补充页面，要么从菜单移除。

---

### 12. 超大视图组件未拆分

| 文件 | 行数 | 体积 |
|---|---|---|
| `counselor/Interventions.vue` | **1599 行** | 46 KB |
| `counselor/Warnings.vue` | **1223 行** | 35 KB |

### 13. 四个 Layout CSS 大量重复

`Layout.vue`、`TeacherLayout.vue`、`CounselorLayout.vue`、`AdminLayout.vue` 各自维护了几乎完全相同的侧边栏样式（约 150 行 CSS / 文件），总计约 600 行重复样式。

**建议**：提取 `BaseLayout.vue`，各角色 Layout 只覆盖菜单项和主题色差异。

`forward/` 目录下没有 `.gitignore`，意味着 `node_modules/`、`dist/`、`.env.local` 等文件有被意外提交的风险。

**建议**：添加标准 Vue 项目的 `.gitignore`。
