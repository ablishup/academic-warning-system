## 三、剩余问题（非阻塞，可逐步优化）

### 1. 后端视图风格仍不统一

当前后端混用 3 种视图编写方式：

| 应用 | 风格 |
|---|---|
| classes, users | @api_view 装饰器函数 |
| algorithm, teacher_dashboard | APIView 子类 |
| courses, import_app, interventions, learning, warning_system | APIView + generics 混用 |

**影响**：代码风格不一致，增加后续维护成本。但当前所有接口功能正常。\
**建议**：毕业答辩前不碰，后续迭代时新接口统一使用 ModelViewSet。

---

### 2. teacher_dashboard/models.py 为空壳

文件仅包含默认注释（60 字节），无实际模型定义。

**影响**：该应用有视图和 URL 路由，但无数据模型，所有数据可能来自其他应用的表或原始 SQL 查询。\
**建议**：如果该应用确实不需要模型，可将 models.py 保留为空或删除；如果需要存储教师Dashboard相关数据，补充模型定义。

---

### 3. 前端调试代码与 mock 数据

- console.log / console.error：57 处
- mock / 模拟数据：11 处

**影响**：开发阶段方便调试，但生产环境会暴露内部数据结构和状态。\
**建议**：答辩前全局搜索 console.log 并批量删除；mock 数据建议仅在 import.meta.env.DEV 时启用。

---

### 4. 超大视图组件未拆分

| 文件 | 行数 | 体积 |
|---|---|---|
| counselor/Interventions.vue | 1599 行 | 46 KB |
| counselor/Warnings.vue | 1223 行 | 35 KB |
| student/Analysis.vue | 896 行 | 24 KB |

**影响**：维护困难，单个文件承担列表、表单、弹窗、图表等多重职责。\
**建议**：时间允许时按职责拆分为子组件，但当前功能正常，不阻塞答辩。

---

### 5. 四个 Layout CSS 大量重复

Layout.vue、TeacherLayout.vue、CounselorLayout.vue、AdminLayout.vue 各自维护了约 150 行几乎相同的侧边栏样式，总计约 600 行重复 CSS。

**影响**：如果要调整主题色或布局，需要改 4 个文件。\
**建议**：后续提取 BaseLayout.vue，当前不影响功能。

---

### 6. 测试覆盖不足

- 后端：仅保留 test_prediction.py，其余 tests.py 已清理
- 前端：无任何测试文件

**影响**：缺乏自动化验证，后续迭代容易引入回归 bug。\
**建议**：毕业设计场景下可接受，但如后续继续维护，建议为核心业务接口补充基础测试。

---

### 7. .env 中第三方 API 密钥

.env 文件包含 DEEPSEEK_API_KEY。虽然文件已被 .gitignore 保护，但仍需注意：
- 不要在任何公开场合（邮件、聊天、云盘）分享 .env 文件
- 如果密钥泄露，需在 DeepSeek 控制台立即重置

## 四、生产环境部署 checklist

如果你计划将项目部署到服务器供答辩演示，请确认以下配置：

| 配置项 | 当前 .env 值 | 生产环境建议 |

| VITE_API_BASE_URL | 开发时未设置 | 部署时设置为后端 API 地址 |
