# 记录学习过程

## jwt认证（jason web token）

就是给登录用户令牌，让他一天之内只用调用一次库

```c

场景：10万学生，每人每天操作100次系统

Session模式：
    查库次数 = 10万人 × 100次 = 1000万次/天

JWT模式：
    查库次数 = 10万人 × 1次登录 = 10万次/天
    
    即使每次登录查库成本高10倍（JOIN多表）
    总成本仍是：10万 × 10 = 100万单位  vs  1000万单位
```

| 误区          | 正确理解               |
| ----------- | ------------------ |
| JWT完全不用数据库  | ❌ 登录时要用，只是请求时不用    |
| 查库就是浪费      | ❌ 重复查相同数据才是浪费      |
| Payload应该极小 | ❌ 应该"静态且高频需用的"都放进去 |
