前提：本项目在开发过程中使用 Claude Code (AI 编程 Agent) 辅助构建，以下是 Agent 驱动开发的具体成果。

一、Agent 驱动的核心成果

1. 双数据源智能整合
   Agent 分析了公网数据(1513人/20门课)和雨课堂数据(40581人/3888门课)的结构差异，通过姓名+login_name双重匹配建立数据源桥梁，从554+206+135人重叠池中精确采样390名学生，每个人2门课100%使用真实学习数据。

2. 随机森林预警模型
   Agent 实现了10维特征向量特征工程(出勤率/视频进度/作业均分/作业提交率/考试均分等)，训练随机森林分类器(准确率96.15%)，按数据分布自适应调整风险阈值(<65高危/<75中危/<80低危)。

3. 数据库分数标准化
   Agent 发现"数据库原理"课程评分尺度异常(均值53.9 vs 其他5门课72-73)，自动执行z-score标准化映射到统一分布，消除评分标准不一对模型的影响。

4. 全栈安全加固
   Agent 识别并修复：SECRET_KEY/数据库密码硬编码→环境变量、CORS_ALLOW_ALL→条件配置、零路由守卫→beforeEach角色越权拦截、Pinia空转→JWT auth store、5处裸except→显式异常捕获。

5. 项目结构重构
   Agent 删除了10个空壳Django应用、2套废弃SQL方案、重复干预模块、8个空tests.py，提取2个公共函数消除4处代码重复，精简requirements.txt从278行→14行。

二、核心逻辑流

单链推理管道(非多Agent协作)：
多源数据采集 → 特征工程(加权融合:出勤30%+作业30%+视频20%+考试20%) → 随机森林模型分类 → 四级风险判定 → 预警记录生成 → 辅导员干预 → 效果评估反馈
