# 后端数据对接与存储说明

## 一、数据存在哪里

```
back/backend/db.sqlite3    ← 唯一的运行时数据库
```

Django 通过 ORM 操作 SQLite，无需安装 MySQL。`settings.py` 中虽有 MySQL 配置但实际未启用——`pymysql.install_as_MySQLdb()` 在连接失败时会静默回退，系统实际运行在 SQLite 上。

## 二、用户角色体系

### 2.1 核心表关系

```
auth_user (Django 内置)
  id, username, password, is_superuser, is_active ...

  ├── Teacher (1:1)  via teachers.user_id → auth_user.id
  │     user_id, teacher_no, department, title
  │
  └── Counselor (1:1) via counselors.user_id → auth_user.id
        user_id, employee_no, department
```

**角色判定逻辑**（`UserSerializer.get_role()`）：
```
is_superuser → 'admin'
groups 有 'teacher' → 'teacher'
groups 有 'counselor' → 'counselor'
以上都不满足 → 'student'
```

## 三、辅导员 → 班级 → 学生 关联链

### 3.1 数据流

```
Counselor (counselors.user_id)
    │
    └── Class (classes.counselor_id = auth_user.id)
          │   ← 注意：counselor_id 存的是 User.id，不是 Counselor.id
          │
          └── Student (students.class_id = classes.id)
```

| counselor3 | 74 | 2021级 | 计算机科学与技术1/2/3班 |

### 3.3 查询代码

```python
# users/utils.py
def get_counselor_student_ids(user):
    """辅导员 → 管理的班级 → 所有学生ID"""
    managed_classes = Class.objects.filter(counselor_id=user.id)
    class_ids = [c.id for c in managed_classes]
    return list(Student.objects.filter(class_id__in=class_ids)
                .values_list('id', flat=True))
```

## 四、教师 → 课程 → 学生 关联链

### 4.1 数据流

```
Teacher (teachers.user_id)
    │
    └── Course (courses.teacher_id = teachers.user_id)

          └── CourseEnrollment (course_enrollments)
```

## 五、学生 → 课程 → 预警 关联链

### 5.1 数据流

```
Student
    │
    ├── CourseEnrollment (选课关系)
    │     student_id + course_id (联合唯一)
    │
    ├── StudentCourseScore (学习指标聚合)
    │     attendance_rate, video_progress, homework_avg,
    │     homework_submit_rate, exam_avg, knowledge_mastery
    │     unique(student_id, course_id)
    │
    └── WarningRecord (预警记录)
          student_id + course_id
          risk_level: high/medium/low/normal
          composite_score, 各维度得分
```

### 5.2 当前数据

- **390学生** × 2课程 = **780条** StudentCourseScore
- **780条** WarningRecord（与Score一一对应）
- 模型准确率: 96.15%

## 六、学生账号 → 学生数据 的匹配

学生用户登录后需要定位到自己的 Student 记录。逻辑在 `get_student_from_user()`：

```
方法1: username 匹配 student_no（学号）
方法2: first_name 匹配 name（姓名）
方法3: 兼容旧账号 student_N
方法4: profile.employee_no 匹配 student_no
```

当前系统的390个学生**没有对应的 Django User 账号**（旧账号已清理），学生端登录需创建 student 账号并绑定 student_no。

## 七、干预记录 关联链

```
WarningRecord (预警触发)
    │
    └── InterventionRecord (干预记录)
          student_id → Student
          course_id → Course (可选)
          warning_id → WarningRecord (可选)
          intervenor_id → auth_user (辅导员)
          intervention_type: talk/academic/psychological/family/other
          is_effective: 0无效/1有效/2待评估
```

## 八、关键关联速查表

| 关联关系 | 字段 | 说明 |
|---------|------|------|
| User ⇄ Teacher | teachers.user_id = auth_user.id | 1:1 |
| User ⇄ Counselor | counselors.user_id = auth_user.id | 1:1 |
| Counselor → Class | classes.counselor_id = auth_user.id | 存 User.id 非 Counselor.id |
| Class → Student | students.class_id = classes.id | 1:N |
| Teacher → Course | courses.teacher_id = teachers.user_id | 1:N |
| Student ⇄ Course | course_enrollments (student_id, course_id) | M:N |
| Student+Course → Score | student_course_scores (student_id, course_id) | 1:1 |
| Score → Warning | warning_records (student_id, course_id) | 1:1 |
| Warning → Intervention | intervention_records.warning_id | 1:N |
